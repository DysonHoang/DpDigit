01 · Khái niệm

Digit DP là gì?

Digit DP là kỹ thuật QHĐ dùng để đếm số nguyên trong đoạn [L, R] thỏa mãn một điều kiện nào đó liên quan đến từng chữ số của số đó.

Thay vì duyệt tuần tự từng số (O(R) → quá chậm), ta xây dựng số từng chữ số một từ trái sang phải và dùng memoization để tránh tính lại các trạng thái giống nhau.

💡 Bài toán kinh điển: "Đếm số nguyên trong [1, N] có tổng chữ số chia hết cho K."

03 · Cấu trúc chung

Template C++

// Bài toán: đếm x ∈ [0, N] sao cho sum chữ số của x bằng S

string N;
int dp[18][162][2];  // dp[pos][sum][tight]
bool visited[18][162][2];

int solve(int pos, int sum, bool tight) {
    if (pos == N.size()) {
        return (sum == S) ? 1 : 0;  // base case
    }
    if (visited[pos][sum][tight])
        return dp[pos][sum][tight];

    visited[pos][sum][tight] = true;
    int limit = tight ? (N[pos] - '0') : 9;
    int res = 0;

    for (int d = 0; d <= limit; d++) {
        // tight mới = tight cũ AND d == limit
        res += solve(pos+1, sum+d, tight && d==limit);
    }

    return dp[pos][sum][tight] = res;
}

// Đếm x ∈ [L, R]:  count(R) - count(L-1)
int ans = solve(0, 0, true);

C++17
⚠️ Trick f(R) − f(L−1): Thay vì viết hàm cho đoạn [L,R], ta viết count(N) = đếm số thỏa điều kiện trong [0,N], rồi trả lời = count(R) − count(L−1).
04 · Quy trình

5 bước giải Digit DP

1

Xác định trạng thái

Tìm những thông tin cần lưu: vị trí đang xét, tình trạng tight, và các tham số bổ sung (tổng chữ số, giá trị mod K, số lần xuất hiện của chữ số nào đó…)

2

Viết hàm đệ quy + memo

Hàm solve(pos, ...state, tight) trả về số lượng số hợp lệ có thể xây tiếp từ vị trí pos với trạng thái hiện tại.

3

Xử lý base case

Khi pos == len: kiểm tra trạng thái tích lũy có thỏa điều kiện không → trả về 0 hoặc 1.

4

Duyệt chữ số d từ 0 đến limit

limit = tight ? N[pos] : 9. Với mỗi d, cập nhật state và gọi đệ quy tiếp theo với new_tight = tight && (d == limit).

5

Áp dụng công thức count(R) − count(L−1)

Xử lý đoạn [L, R] bằng cách gọi hàm hai lần và trừ kết quả.

05 · Ví dụ đầy đủ

Đếm số trong [1,N] không chứa chữ số 4

#include <bits/stdc++.h>
using namespace std;

string N;
map<pair<int,int>, long long> memo;

// pos: vị trí đang xét
// tight: có bị ràng buộc bởi N không
// leading: có đang ở phần leading zeros không
long long dp(int pos, bool tight, bool leading) {
    if (pos == (int)N.size()) return 1;

    auto key = make_pair(pos, tight);
    if (!leading && memo.count(key)) return memo[key];

    int limit = tight ? (N[pos] - '0') : 9;
    long long res = 0;

    for (int d = 0; d <= limit; d++) {
        if (d == 4) continue;  // bỏ qua chữ số 4
        res += dp(pos+1, tight && d==limit, leading && d==0);
    }

    if (!leading) memo[key] = res;
    return res;
}

int main() {
    cin >> N;
    // Trả về count trong [0,N], trừ 1 cho số 0
    cout << dp(0, true, true) - 1;
}

06 · Phân tích

Độ phức tạp

O(D)
Số chữ số của N (thường ≤ 18)
O(D × S)
Số trạng thái (D × kích thước state)
× 10
Mỗi trạng thái xét 10 chữ số

Với N ≤ 10¹⁸ (18 chữ số), tổng số trạng thái thường rất nhỏ → Digit DP cực kỳ hiệu quả.

Total states = D × |state_1| × |state_2| × ... × 2 (tight)
07 · Lưu ý quan trọng

Leading Zeros

Khi xây số "0 0 7", hai số 0 đầu là leading zeros — không phải chữ số thực sự của số 7. Nhiều bài toán cần phân biệt trường hợp này (ví dụ: đếm số chữ số 0 thực sự).

Giải pháp: thêm flag leading_zero = true khi chưa đặt chữ số khác 0 nào. Khi leading_zero là true, không tính vào state và không memo (vì trạng thái không ổn định).
